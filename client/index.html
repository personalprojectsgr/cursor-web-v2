Get Started with Cursor Web

Control your Cursor IDE remotely from any device

1

Launch Cursor with Remote Debugging

Start Cursor IDE with the CDP flag so this app can connect to it.

Terminal 
cursor --remote-debugging-port=9222
Windows shortcut tip

Right-click your Cursor desktop shortcut, go to Properties, and append the flag to the Target field:

"C:\Users\...\Cursor.exe" --remote-debugging-port=9222
2

Configure MCP for the Conversation Loop

Add the MCP server to Cursor so the agent can stay in a persistent loop with you.

~/.cursor/mcp.json 
{
  "mcpServers": {
    "cursor-remote": {
      "url": "http://localhost:3000/mcp"
    }
  }
}
Using Railway instead of localhost?

If your server is deployed on Railway, use the remote URL:

"url": "https://cursorremote.up.railway.app/mcp"
3

Add the Cursor Rule

The agent needs a rule telling it to call wait_for_response after every reply. Create this file in your project:

.cursor/rules/telegram-flow.mdc 
---
description: CursorRemote conversation loop
globs: *
alwaysApply: true
---
You MUST call wait_for_response from the
cursor-remote MCP server as the VERY LAST
action of EVERY response. No exceptions.
If it returns empty or errors, call it
again immediately. The ONLY exit is /stop
from the user.
4

Start Chatting

Open a chat in Cursor and send any message. The agent will call wait_for_response, the loop will activate, and you can chat from this web interface.
