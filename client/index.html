1

Enable Remote Debugging

Launch Cursor with a CDP port so this app can read its UI.

Terminal 
cursor --remote-debugging-port=9222
Windows: permanent shortcut

Right-click your Cursor shortcut → Properties → append the flag to Target:

"C:\Users\...\Cursor.exe" --remote-debugging-port=9222
2

Add the MCP Server

This lets the agent keep a persistent conversation loop with you.

~/.cursor/mcp.json 
{
  "mcpServers": {
    "cursor-remote": {
      "url": "http://localhost:3000/sse"
    }
  }
}
Deployed on Railway?

Swap the URL for your Railway domain:

"url": "https://cursorremote.up.railway.app/sse"
3

Create the Agent Rule

Tells the agent to call wait_for_response with its workspace name after every reply.

.cursor/rules/cursor-web.mdc 
---
description: CursorRemote conversation loop
alwaysApply: true
---
You MUST call wait_for_response from the
cursor-remote MCP server as the VERY LAST
action of EVERY response. No exceptions.
ALWAYS pass chat_id with your workspace
folder name (e.g. chat_id: "my-project").
If it returns empty, errors, or keepalive,
call it again immediately.
The ONLY exit is /stop from the user.
Why chat_id?

The chat_id parameter lets the server match each agent to its Cursor window. Without it, messages may be delivered to the wrong window when multiple are open.

4

Start a Chat

Open any Cursor chat and send a message. The loop activates, and you can talk from here.
